<클래스>
    클래스란 "객체"를 생성하기 위해 "속성"과 "기능"을 "추상화"한 설계도
    (키워드 기억, 시험에 나올 듯 하다고 하심)
    - 클래스가 필요한 이유 -
    1. 게임을 구성하는 수 많은 객체가 존재
    2. 상호 작용 처리

    <객체 지향>
    게임 구성에 필요한 "로직과 데이터"를 "객체 단위"로 관리
    (플레이어/ 몬스터/ 로비/ 인공지능/ UI 등)

    (레벨 블루프린트 -> 블루프린트 클래스)
    <BP Class>
    (부모 클래스 선택 중..)
    액터 >= 폰 >= 캐릭터 (상속의 개념)

    ( 필요한 이유 )
    형태와 기능은 같지만, 상세 데이터가 다른 경우
    하나의 설계도를 통해 Actor 생성 -> 설계도 수정하나만으로 인스턴스들 일괄 수정 가능

    Make Transform -> Spawn Actor from class (원하는 위치에 Actor 소환)
    Set Actor Transform 노드 -> 자신의 Actor 위치 업데이트

* 폰 자식 클래스를 만들면 해당 클래스의 Actor를 Spawn 할 수 없는 것을 발견했다.
