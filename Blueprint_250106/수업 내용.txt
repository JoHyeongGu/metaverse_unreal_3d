기본 인터페이스 설명 / 조작법

<프레임워크 개념>
1. 월드
    맵 < 월드 (언리얼 엔진 기준 우주, 진리)
    "월드"의 역할
    1) 게임 콘텐츠를 담는 공간 (Transform)
    2) 게임 콘텐츠의 진행 (Tick)
    3) 게임 흐름을 위한 시간 (Time)
    4) 기본 환경 설정 (World Setting)
    5) 월드의 단위 구성 요소 : Actor (Unity에서 GameObject 같은 개념)

2. 모드
    게임 모드의 역할
    1) 플레이어 입장 관리
    2) 게임의 최종 승패 관리
    3) 게임의 규칙 관리
    4) 게임 데이터 검증 (멀티플레이 게임)

3. 기믹
    기믹의 역할
    1) 월드 내 사물과 상호작용 수행
    2) 지정된 조건을 만족하면 이벤트를 발생
    3) 월드에 물체를 스폰(spawn) 시킬 수 있음
    (약간 기능 시나리오같은 개념)

4. 플레이어
    플레이어의 역할
    단순 캐릭터를 말하는 것이 아닌 캐릭터를 조종하는 것에 대한 개념
    (캐릭터는 눈에 보이는 요소 / 플레이어는 눈에 보이는 개념은 아니고 눈에 보이는 것을 기능하게 하는 개념)
    1) 게임에 입장한 사용자와의 1:1 대응
    2) 사용자의 현재 상태 및 데이터 관리
    3) 사용자 입력의 1차 처리
    4) 사용자 화면에 대응하는 카메라 관리
    5) 사용자 화면에 나타나는 HUD UI 관리

5. 폰(Pawn)
    (플레이어에서 좀 더 기능 컨트롤적인 느낌의 개념)
    폰의 역할
    1) 빙의(Possess)를 통해 플레이어와 연결
    2) 사용자 입력의 실제 처리
    3) 사용자 화면에 대응되는 카메라 설정
    4) 기믹과의 상호작용
    5) 현재 상태에 적합한 애니메이션 재생

플레이어와 폰(아바타)의 차이점?
    플레이어는 영혼, 폰은 육체
    플레이어는 눈에 안 보이는 거, 폰은 눈에 보이는 거(물리적) 
    아바타가 폰이고 플레이어가 기능 개념
