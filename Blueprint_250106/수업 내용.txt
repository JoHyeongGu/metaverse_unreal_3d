기본 인터페이스 설명 / 조작법

<언리얼 프레임워크 개념>
1. 월드
    맵 < 월드 (언리얼 엔진 기준 우주, 진리)
    "월드"의 역할
    1) 게임 콘텐츠를 담는 공간 (Transform)
    2) 게임 콘텐츠의 진행 (Tick)
    3) 게임 흐름을 위한 시간 (Time)
    4) 기본 환경 설정 (World Setting)
    5) 월드의 단위 구성 요소 : Actor (Unity에서 GameObject 같은 개념)

2. 모드
    게임 모드의 역할
    1) 플레이어 입장 관리
    2) 게임의 최종 승패 관리
    3) 게임의 규칙 관리
    4) 게임 데이터 검증 (멀티플레이 게임)

3. 기믹
    기믹의 역할
    1) 월드 내 사물과 상호작용 수행
    2) 지정된 조건을 만족하면 이벤트를 발생
    3) 월드에 물체를 스폰(spawn) 시킬 수 있음
    (약간 기능 시나리오같은 개념)

4. 플레이어
    플레이어의 역할
    단순 캐릭터를 말하는 것이 아닌 캐릭터를 조종하는 것에 대한 개념
    (캐릭터는 눈에 보이는 요소 / 플레이어는 눈에 보이는 개념은 아니고 눈에 보이는 것을 기능하게 하는 개념)
    1) 게임에 입장한 사용자와의 1:1 대응
    2) 사용자의 현재 상태 및 데이터 관리
    3) 사용자 입력의 1차 처리
    4) 사용자 화면에 대응하는 카메라 관리
    5) 사용자 화면에 나타나는 HUD UI 관리

5. 폰(Pawn)
    (플레이어에서 좀 더 기능 컨트롤적인 느낌의 개념)
    폰의 역할
    1) 빙의(Possess)를 통해 플레이어와 연결
    2) 사용자 입력의 실제 처리
    3) 사용자 화면에 대응되는 카메라 설정
    4) 기믹과의 상호작용
    5) 현재 상태에 적합한 애니메이션 재생

플레이어와 폰(아바타)의 차이점?
    플레이어는 영혼, 폰은 육체
    플레이어는 눈에 안 보이는 거, 폰은 눈에 보이는 거(물리적) 
    아바타가 폰이고 플레이어가 기능 개념

<언리얼 블루프린트 기초>
1. 블루프린트 구성요소
    1) 순서도
    2) 논리 표현
    (노트) =(제어의 흐름)=> (노드)
    (출력핀) =(데이터의 흐름)=> (입력핀)
2. 실습: 레벨 블루프린트를 사용해보자
    레벨 블루프린트 열기
    노드 생성 / 코멘트 생성 / 노드 연결하기
    키보드 1,2,3 입력 노드 생성 / 입력에 따른 출력 노드 생성
    변수 타입 지정 및 생성 / 변수 Get, Set 노드 생성 (타입별로 다른 Set 노드)
    "네임, 스트링, 텍스트 모두 문자열 저장"
        네임 - 텍스트가 바뀔 일이 없을 때, (상수 저장)
        스트링 - 값이 바뀌는 문자열 (내가 알고 있는 string 타입)
        텍스트 - 값이 바뀔 수 있고 로컬라이징이 필요할 때 사용하는 타입
        로컬라이징(현지화): 키 값에 따라 텍스트가 바뀌어서 언어에 따라 다르게 할 수 있음
    예시) 마우스 왼쪽 클릭 시 총알을 발사, 남은 총알의 수(Ammo)를 표시한다.
